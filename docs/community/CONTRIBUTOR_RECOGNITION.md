# Contributor Recognition

AIByDM treats documentation, content, design, triage, testing, and community work as real contributions.

## Recognition Channels

| Channel | When to use it |
| --- | --- |
| Release notes | Meaningful feature, content, docs, or community contributions. |
| Changelog | User-facing platform changes, contributor workflow changes, and release operations. |
| Community docs | Sustained contributors, maintainers, and milestone contributors. |
| GitHub contributors graph | All merged commits. |
| Discussions | Launch notes, contributor spotlights, and calls for help. |

## Maintainer Checklist

1. Mention meaningful contributor work in release notes.
2. Keep `CHANGELOG.md` updated for platform changes.
3. Thank first-time contributors in pull request follow-up.
4. Add recurring contributors to area ownership discussions when appropriate.
5. Keep recognition tied to public work in the repository.
