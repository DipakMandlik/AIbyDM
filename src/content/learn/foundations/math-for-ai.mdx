## Overview

Mathematics is the language that AI is written in. While modern frameworks abstract away much of the computation, understanding the underlying math gives you the ability to debug models, design new architectures, interpret results, and read research papers. This module covers the four mathematical pillars of AI: linear algebra, calculus, probability, and statistics.

You do not need a mathematics degree to succeed in AI. You need working fluency in specific concepts that appear repeatedly across every subfield -- from classical machine learning to transformer-based language models. This lesson focuses on exactly those concepts and nothing more.

## Theory

### Pillar 1: Linear Algebra

Linear algebra is the mathematics of data. Every dataset, every image, every embedding is a matrix or a tensor. Neural networks are chains of matrix multiplications.

#### Vectors

A vector is an ordered list of numbers. In AI, vectors represent data points, features, weights, and gradients.

```python
import numpy as np

# A feature vector for a single data point
# [height_cm, weight_kg, age_years]
person = np.array([175, 70, 30])

# Vector operations
a = np.array([1, 2, 3])
b = np.array([4, 5, 6])

addition = a + b           # [5, 7, 9]
scalar_mult = 3 * a        # [3, 6, 9]
dot_product = np.dot(a, b) # 1*4 + 2*5 + 3*6 = 32
magnitude = np.linalg.norm(a)  # sqrt(1 + 4 + 9) = 3.742
```

The **dot product** appears everywhere in AI. When a neural network computes the output of a neuron, it takes the dot product of the input vector with the weight vector. When a search engine ranks documents, it computes the dot product (or cosine similarity) between a query embedding and document embeddings.

#### Matrices and Matrix Multiplication

A matrix is a 2D array of numbers. A dataset with 1000 samples and 10 features is a 1000x10 matrix.

```python
# Dataset: 3 samples, 4 features
X = np.array([
    [1.0, 0.5, 0.3, 0.8],
    [0.2, 0.9, 0.7, 0.1],
    [0.6, 0.4, 0.5, 0.3],
])

# Weight matrix: 4 inputs -> 2 outputs
W = np.array([
    [0.1, 0.4],
    [0.3, 0.2],
    [0.5, 0.1],
    [0.2, 0.3],
])

# Forward pass: matrix multiplication
output = X @ W  # shape: (3, 2)
# Each row of the output is the transformed representation of one sample
```

Matrix multiplication is the most performance-critical operation in deep learning. GPUs are purpose-built to perform thousands of these operations in parallel.

#### Transpose, Inverse, and Eigenvalues

```python
# Transpose -- swap rows and columns
X_T = X.T  # shape changes from (3, 4) to (4, 3)

# Inverse -- for square matrices only
A = np.array([[2, 1], [5, 3]])
A_inv = np.linalg.inv(A)
# A @ A_inv should give the identity matrix

# Eigenvalues -- critical in PCA (dimensionality reduction)
cov_matrix = np.cov(X.T)
eigenvalues, eigenvectors = np.linalg.eig(cov_matrix)
```

Eigenvalues tell you how much variance each direction in your data captures. Principal Component Analysis (PCA) uses eigendecomposition to reduce the dimensionality of data while preserving as much information as possible.

### Pillar 2: Calculus

Calculus answers the question: "How does the output change when I change the input?" In AI, this question is the foundation of learning -- a model learns by measuring how its error changes with respect to each weight, then adjusting those weights to reduce the error.

#### Derivatives and Gradients

The derivative of a function tells you the rate of change at any point. The gradient is the multi-dimensional extension -- it tells you the direction of steepest increase for a function with multiple inputs.

```python
# Conceptual: derivative of f(x) = x^2
# f'(x) = 2x
# At x=3, the slope is 6 -- the function is increasing steeply

# In AI, we compute gradients of the loss with respect to weights
# If loss = (prediction - target)^2
# d(loss)/d(prediction) = 2 * (prediction - target)

def compute_gradient(prediction, target):
    """Gradient of MSE loss with respect to prediction."""
    return 2 * (prediction - target)

pred = 0.8
target = 1.0
grad = compute_gradient(pred, target)
# grad = -0.4 (negative means we should increase the prediction)
```

#### The Chain Rule and Backpropagation

The chain rule lets you compute derivatives of composed functions. Since a neural network is a composition of many functions (layer after layer), the chain rule is what makes training possible.

```python
# A simple 2-layer network: input -> hidden -> output
# forward: h = sigmoid(x * w1), y = h * w2
# The chain rule gives us:
# d(loss)/d(w1) = d(loss)/d(y) * d(y)/d(h) * d(h)/d(w1)

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def sigmoid_derivative(x):
    s = sigmoid(x)
    return s * (1 - s)
```

Backpropagation is just the chain rule applied systematically from the output layer back to the input layer. Modern frameworks like PyTorch compute this automatically, but understanding it lets you debug gradient issues like vanishing or exploding gradients.

#### Gradient Descent

Gradient descent is the optimization algorithm that uses gradients to update model weights.

```python
def gradient_descent(X, y, lr=0.01, epochs=1000):
    """Train a simple linear model using gradient descent."""
    n_features = X.shape[1]
    weights = np.zeros(n_features)
    bias = 0.0

    for epoch in range(epochs):
        # Forward pass
        predictions = X @ weights + bias

        # Compute loss (MSE)
        error = predictions - y
        loss = np.mean(error ** 2)

        # Compute gradients
        grad_w = (2 / len(y)) * (X.T @ error)
        grad_b = (2 / len(y)) * np.sum(error)

        # Update weights (move opposite to gradient)
        weights -= lr * grad_w
        bias -= lr * grad_b

        if epoch % 200 == 0:
            print(f"Epoch {epoch}: loss = {loss:.6f}")

    return weights, bias
```

### Pillar 3: Probability

Probability quantifies uncertainty. AI systems operate in the real world where data is noisy, incomplete, and ambiguous. Probability gives us the framework to reason about this uncertainty.

#### Core Concepts

- **Probability** P(A) -- the likelihood of event A, between 0 and 1.
- **Conditional probability** P(A|B) -- the probability of A given that B has occurred.
- **Bayes' theorem** -- P(A|B) = P(B|A) \* P(A) / P(B) -- lets you update beliefs with new evidence.
- **Independence** -- P(A and B) = P(A) \* P(B) when events do not influence each other.

```python
# Bayes' theorem in practice: spam filtering
# P(spam | word="free") = P("free" | spam) * P(spam) / P("free")

p_free_given_spam = 0.80   # 80% of spam emails contain "free"
p_spam = 0.30              # 30% of all emails are spam
p_free = 0.35              # 35% of all emails contain "free"

p_spam_given_free = (p_free_given_spam * p_spam) / p_free
print(f"P(spam | 'free') = {p_spam_given_free:.2f}")
# Result: 0.69 -- a 69% chance the email is spam
```

#### Probability Distributions

Distributions describe the range and likelihood of values a random variable can take.

- **Normal (Gaussian)** -- the bell curve. Weight initialization in neural networks often samples from a normal distribution.
- **Uniform** -- all values equally likely. Used for random search in hyperparameter tuning.
- **Bernoulli** -- binary outcome (0 or 1). Models a single yes/no decision.
- **Softmax distribution** -- converts raw model outputs (logits) into probabilities that sum to 1.

```python
# Softmax -- converts logits to probabilities
def softmax(logits):
    exp_values = np.exp(logits - np.max(logits))  # subtract max for stability
    return exp_values / exp_values.sum()

logits = np.array([2.0, 1.0, 0.1])
probs = softmax(logits)
print(f"Probabilities: {probs.round(3)}")
# [0.659, 0.242, 0.099] -- highest logit gets highest probability
```

### Pillar 4: Statistics

Statistics gives you the tools to understand data, detect patterns, and validate model performance.

#### Descriptive Statistics

```python
data = np.random.normal(loc=50, scale=10, size=1000)

mean = np.mean(data)        # central tendency
median = np.median(data)    # robust to outliers
std = np.std(data)          # spread of data
variance = np.var(data)     # std squared

# Correlation -- do two variables move together?
feature_a = np.random.randn(100)
feature_b = feature_a * 0.8 + np.random.randn(100) * 0.5
correlation = np.corrcoef(feature_a, feature_b)[0, 1]
print(f"Correlation: {correlation:.3f}")
# Close to 1.0 means strong positive relationship
```

#### Hypothesis Testing and Confidence

In AI, you constantly compare models. Is Model A truly better than Model B, or was the difference due to random variation?

```python
from scipy import stats

# Two models evaluated on the same test set
model_a_scores = np.array([0.91, 0.89, 0.93, 0.90, 0.92])
model_b_scores = np.array([0.88, 0.87, 0.89, 0.86, 0.88])

# Paired t-test: is the difference statistically significant?
t_stat, p_value = stats.ttest_rel(model_a_scores, model_b_scores)
print(f"t-statistic: {t_stat:.3f}, p-value: {p_value:.4f}")
# If p_value < 0.05, the difference is likely real, not random chance
```

## Key Takeaways

- **Linear algebra** (vectors, matrices, dot products) is the language of data representation and neural network computation.
- **Calculus** (derivatives, chain rule, gradient descent) is the engine that makes learning from data possible.
- **Probability** (Bayes' theorem, distributions, softmax) lets AI systems reason under uncertainty and output calibrated predictions.
- **Statistics** (descriptive stats, hypothesis testing) gives you the tools to understand data and rigorously evaluate model performance.
- You do not need to master all of this before starting -- build working fluency in each area and deepen your knowledge as you encounter these concepts in real projects.
