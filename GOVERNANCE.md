# Governance

AIByDM is maintained as an open source learning project with a lightweight,
transparent decision process.

## Roles

| Role         | Responsibility                                                           |
| ------------ | ------------------------------------------------------------------------ |
| Maintainers  | Project direction, releases, repository settings, final review decisions |
| Reviewers    | Review content, tools, exams, docs, and code in their area of expertise  |
| Contributors | Open issues, submit PRs, improve docs, content, and features             |

## Decision Making

- Small fixes can be merged after CI passes and the required review is complete.
- Larger changes should begin as a GitHub Discussion or issue before a PR.
- Maintainers make final calls when there is disagreement, prioritizing learner
  value, accuracy, accessibility, and maintainability.

## Review Rules

- Content PRs require at least 1 approving review.
- Code, CI, and architecture PRs require at least 1 maintainer approval.
- Security-sensitive changes require maintainer review before merge.
- Changes must pass the repository validation workflow before merge.

## Becoming a Reviewer or Maintainer

Regular contributors may be invited to review or maintain specific areas after
consistent, high-quality participation. Maintainers should update
[MAINTAINERS.md](MAINTAINERS.md) when responsibilities change.
